# CRM для учебных центров - Frontend

React приложение для управления учебным процессом с поддержкой нескольких ролей пользователей.

## Стек технологий

- **React 18** - UI библиотека
- **TypeScript** - типобезопасность
- **Vite** - быстрая сборка
- **React Router** - маршрутизация
- **Zustand** - управление состоянием
- **Tailwind CSS** - стилизация
- **Lucide React** - иконки

## Структура проекта

```
frontend/
├── src/
│   ├── components/     # Переиспользуемые компоненты
│   │   ├── common/     # Общие компоненты
│   │   ├── forms/      # Формы
│   │   └── layout/     # Layout компоненты (Sidebar, Header)
│   ├── pages/          # Страницы приложения
│   │   ├── auth/       # Аутентификация (Login, Register)
│   │   ├── admin/      # Страницы админа
│   │   ├── instructor/ # Страницы преподавателя
│   │   ├── student/    # Страницы студента
│   │   └── common/     # Общие страницы
│   ├── context/        # State management (Zustand)
│   ├── services/       # API сервисы
│   ├── types/          # TypeScript типы
│   ├── utils/          # Утилиты
│   ├── styles/         # Глобальные стили
│   └── assets/         # Статические файлы
├── public/             # Публичные файлы
└── package.json
```

## Роли пользователей

### Super Admin / Admin
- Приглашение пользователей
- Управление всеми пользователями
- Просмотр статистики системы
- Управление курсами и студентами

### Instructor (Преподаватель)
- Создание и управление курсами
- Управление расписанием занятий
- Прикрепление материалов и записей
- Создание домашних заданий
- Проверка работ студентов
- Просмотр успеваемости

### Student (Студент)
- Просмотр календаря занятий
- Доступ к материалам и записям
- Сдача домашних заданий
- Просмотр оценок и успеваемости
- AI ассистент для помощи в обучении
- Управление оплатой через Kaspi Pay

## Установка и запуск

### Требования
- Node.js 18+
- npm или yarn

### Установка зависимостей

\`\`\`bash
npm install
\`\`\`

### Запуск в режиме разработки

\`\`\`bash
npm run dev
\`\`\`

Приложение будет доступно по адресу: http://localhost:3000

### Сборка для продакшена

\`\`\`bash
npm run build
\`\`\`

### Предпросмотр production сборки

\`\`\`bash
npm run preview
\`\`\`

## Конфигурация

### Proxy для API

В `vite.config.ts` настроен proxy для API запросов:
- `/api` -> `http://localhost:8080` (Spring Boot backend)

Когда бэкенд будет готов, все запросы к `/api/*` будут автоматически проксироваться.

## Текущий статус

На данный момент фронтенд работает с моковыми данными. После готовности Spring Boot бэкенда:

1. Обновить API сервисы в `src/services/`
2. Подключить реальные эндпоинты
3. Настроить обработку ошибок
4. Добавить загрузочные состояния

## Особенности реализации

- **Адаптивный дизайн** - работает на всех устройствах
- **Type-safe** - полная типизация с TypeScript
- **Модульная структура** - легко расширяемая архитектура
- **Role-based routing** - маршруты в зависимости от роли пользователя
- **Готовность к интеграции** - структура под API-first подход
